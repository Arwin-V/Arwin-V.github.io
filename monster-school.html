Monster School

Unreal Engine 5.4 Blueprint Visual Scripting Wave-Based FPS Object Pooling AI State Machines
ROLE: Programmer

Core Engineering Implementations

  1. Optimized First-Person Architecture: Engineered a high-performance FPS controller utilizing deterministic line-tracing for hit registration. Minimized render overhead by exclusively drawing a first-person arm mesh, paired with procedural recoil animations for tactile weapon feedback.
  2. Memory-Efficient Object Pooling: Architected a robust object pooling system for projectile instantiation and VFX. This completely eliminates runtime garbage collection hitches and frame drops during high-density combat encounters.
  3. State-Driven AI & Dynamic Target Tracking: Programmed autonomous drone AI utilizing Unreal's navigation mesh and state machines. Implemented seamless transitions between patrol and pursuit states, natively bound to Dynamic Material Instance (MID) updates (e.g., threat-level color shifting to red) to visually telegraph AI intent.
  4. Wave Management & Modular Timers: Developed a scalable, wave-based encounter director driven by a modular timing subsystem. This manages dynamic pacing, spawning varied monster and drone archetypes across the abandoned city environment.
  5. Decoupled Damage & Resource Framework: Built a modular, component-based damage system handling localized hit calculations. Integrated fluid player locomotion (sprint, jump) with resource management loops (health and ammo pickups) directly into the core combat loop.
  6. Observer Pattern UI Architecture: Bound the player HUD strictly to data-driven event dispatchers. By adopting the Observer pattern, the UI updates purely on state mutation (health/ammo changes), completely avoiding expensive per-frame ticking.
  7. Spatial Audio Engineering: Integrated procedural and spatialized audio cues to enhance threat localization, elevating the acoustic feedback loop for enemy spawns, drone proximity, and weapon mechanics.
