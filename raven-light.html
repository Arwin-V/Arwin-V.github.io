System Architecture

High-level execution flow demonstrating decoupled state persistence, interface-driven transformations, and event-driven AI.

Core Engineering Implementations

  1. State Management & Memory Persistence: Abstracted RPG data (Health, Stamina, Mana) into a modular UCustomAttributeComponent bound to the APlayerState. Maintained strict data integrity across boundaries by leveraging UGameInstance caches for hard loads and persistent state injection during seamless transitions.
  2. World Streaming & Seamless Transitions: Architected non-blocking, seamless level transitions utilizing Unreal’s level streaming and portal systems. This completely eliminates frame hitching and ensures uninterrupted gameplay flow during spatial state changes.
  3. Interface-Driven Possession Pipeline: Engineered a scalable ITransformable framework to decouple pawn logic. The pipeline evaluates Hierarchical FGameplayTag conditions, safely handles the garbage collection of the active shell, and dynamically injects Priority 0 Input Mapping Contexts (IMCs) upon target possession.
  4. Scalable Combat Architecture & Token Director: Designed a highly scalable, component-based combat framework governed by a custom Token-Based Attack System. This architecture manages AI attack concurrency and spacing, preventing combat state overlapping and guaranteeing readable, AAA-standard engagement pacing.
  5. Deterministic Hit Registration (Advanced Trace): Developed a universal, data-driven UAdvancedWeaponTraceComponent attachable to any entity via composition. Hit calculation is strictly slaved to UAnimNotifyState events within Montages, guaranteeing frame-perfect synchronization between visual kinematics and server-side damage execution.
  6. Event-Driven AI & Bespoke Boss Logic: Replaced expensive tick functions with delegate-driven UBehaviorTree execution for standard AI. Architected a bespoke, standalone AI Controller for the Boss entity to manage complex phase transitions and strictly mitigate "God Class" inheritance bloat.
  7. Procedural 3D Spatial Navigation: Programmed a custom UProceduralFlightComponent to solve complex aerial traversal for entities like Dragons. Combined spline-based locomotion with dynamic steering behaviors (e.g., separation, alignment, cohesion) to generate fluid, procedural flight animations independently of the primary navigation mesh.
  8. Data-Driven Design (DDD) & Observer UI: Constructed the ABirdBase architecture heavily reliant on UPrimaryDataAsset initialization, enabling level designers to natively tune parameters without C++ recompilation. Bound the UI strictly to OnValueChanged delegates, achieving zero-tick, Observer Pattern-driven rendering.
